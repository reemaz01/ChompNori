# ChompNori BiteBuddy - Asynchronous Database Layer Integration

ChompNori BiteBuddy is a cyber-minimal and warm-light nutrition dashboard. To elevate its reliability, scalability, and user experience, we have successfully replaced all temporary local variables and simple client cache states (`localStorage`) with a production-ready, highly durable, and asynchronous native database engine.

---

## 💾 Database Selection & Architectural Choice

We have implemented **IndexedDB** as the application's primary database layer. 

### Why IndexedDB is the Best Fit:
1. **Asynchronous API**: Operations run inside native transactional promises, ensuring the UI remains 100% smooth, fluid, and responsive even when querying large arrays of nutrient history.
2. **True Structural Data Storage**: Unlike string-based local storage, IndexedDB stores nested JavaScript objects natively with type safety, preventing serialisation performance bottlenecks.
3. **Generous Capacity**: Modern browsers allocate up to hundreds of megabytes (or up to 50% of free disk space) for IndexedDB, ensuring user logged data is never prematurely evicted.
4. **Offline Resilience**: Eliminates the latency, credentials vulnerability, and connection downtime of remote backends, matching the dashboard's self-contained client-side design perfectly.

---

## 🗄️ Database Collections (Object Stores)

BiteBuddy establishes a single robust database named `ChompNoriBiteBuddyDB` with the following object stores:

1. **`profile`**: Stores the user's active biometrics (`UserProfile`), used for custom target formulations.
2. **`entries`**: Stores detailed micro and macro nutrient log records (`LogEntry`) with indexed timestamps.
3. **`water`**: Stores the user's current daily hydration level.
4. **`mealplan`**: Stores the dynamic multi-day dietary recipes and structures generated by BiteBuddy.
5. **`settings`**: Stores general configuration items, including theme modes (Cyber HUD Dark vs Warm Light).

---

## 🛠️ CRUD Operations Implemented

All operations are written using a modular, promisified transaction wrapper in `src/db.ts`:

* **CREATE**:
  * `addEntry(entry)`: Appends new food logs into the database with precise nutrient structures.
  * Native seed triggers populate pre-configured items on initial setup.
* **READ**:
  * `getProfile()`, `getEntries()`, `getWater()`, `getMealPlan()`, `getTheme()`: Asynchronously load complete historical sets.
* **UPDATE**:
  * `updateEntry(entry)`: Allows modifying specific quantities or categories in logged entries.
  * `saveProfile(profile)`: Saves updated physical goals and bio-metrics safely.
* **DELETE**:
  * `deleteEntry(id)`: Removes entries transactionally by key.
  * `handleResetWater()`: Resets hydration metrics.

## 🚀 Setup & Execution

### Prerequisites
None! IndexedDB runs natively on all modern browsers (Chrome, Safari, Firefox, Edge).

### Installation
1. Install base project packages:
   ```bash
   npm install
   ```
2. Launch the development server:
   ```bash
   npm run dev
   ```
3. Build for production:
   ```bash
   npm run build
   ```
